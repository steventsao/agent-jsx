Agent API

The authored API is exported from agent-jsx/agent. Agent files never declare whether they are a parent or child; composition supplies that relationship.

Agent<S, P>

abstract class Agent<
  S extends Record<string, unknown>,
  P extends object = {}
>

A hierarchy-free agent definition. S is durable state and P is the explicit input contract supplied by a composition boundary.

MemberMeaning
initialStateRequired initial durable state.
modelRequired explicit provider/model identifier.
description?Human-readable purpose used by target registries.
displayName?UI label for generated agent/tool surfaces.
propsCurrent typed composition input.

Identity and model

static agentName = "policy-reviewer";
model = "openrouter/anthropic/claude-sonnet-4";

Both values are authored policy. The compiler does not infer provider, model, role, or hierarchy from class names, filenames, or export names.

state and setState(...)

readonly state: S;
setState(next: S | ((state: S) => S)): void;

Read current state and replace it with a value or functional update. Generated Cloudflare targets bridge this operation to Durable Object state; local compilation uses the same store contract.

@callable()

@callable()
approve(input: Approval): Result {
  this.setState(state => applyApproval(state, input));
  return { accepted: true };
}

Marks a method as part of the agent’s explicit public capability surface. Code generation creates the target-specific proxy; undecorated methods remain internal.

getPrompt()

getPrompt() {
  return (
    <prompt>
      <sys p={10}>Review the supplied policy.</sys>
      <msg p={8}>{this.props.policy}</msg>
    </prompt>
  );
}

Returns plain text or prompt JSX. Prompt blocks are rendered under a priority budget. Cloudflare Think maps this to getSystemPrompt(); Flue maps it to instructions.

getTools()

getTools() {
  return {
    lookup: {
      description: "Read one policy by id.",
      execute: ({ id }) => readPolicy(id),
    },
  };
}

Returns an AI SDK-style tool object or declarative <tool> JSX. The compiler lowers the definition into the selected target’s native tool representation.

getSkills()

getSkills() {
  return [releaseNotesSkill, incidentResponseSkill];
}

Returns opaque skill sources owned by the agent. Targets that support skills retain the references without turning them into hierarchy.

render()

render() {
  return <AgentStatus state={this.state} />;
}

Optional UI projection only. The compiler never treats render() as prompt, tools, control flow, or agent hierarchy.

compileAgentClass(...)

const Reviewer = compileAgentClass(PolicyReviewer);

Lowers one authored class into a typed boundary component. Most projects consume generated companion modules and do not call this primitive by hand.

composeAgent(...)

composeAgent(
  <Coordinator name="main">
    {({ task, accept }) => (
      <Worker task={task} onResult={result(accept)} />
    )}
  </Coordinator>
);

Creates a root composition. The function child receives only the root’s getters and callable references. Nested boundaries define runtime hierarchy.

result(callable)

<Worker onResult={result(accept)} />

Explicitly grants a child the authority to return its delegated result to a parent callable. This is a visible capability binding, not a name-based convention.

Generated runtime

Target plumbing is intentionally absent from the authored API:

TargetCompiler output
AgentsDurable classes, state bridges, callable RPC, child props, callbacks, bindings, and migrations.
ThinkgetModel, optional deployment modelResolver, getSystemPrompt, native tools, durable programmatic turns, and text/reasoning traces.
FlueProfiles, instructions, skills, tools, subagent lists, and workflow tasks.
Provider boundary: Model ids stay explicit in agent classes. A Think target may import a deployment-owned modelResolver(env, id) for SDK packages and secrets; neither agent names nor hierarchy select a provider.
Reasoning UI: Think’s model-supplied reasoning parts can be returned by the generated runTurnWithTrace bridge for progress indicators or thought bubbles. Authors do not implement stream callbacks manually.
